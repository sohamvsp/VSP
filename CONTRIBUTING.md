## Discussion Guidelines

### Be Respectful
- Assume good faith
- Be constructive in criticism
- Respect different viewpoints

### Be Clear
- Provide context for your suggestions
- Use technical terminology appropriately
- Link to relevant resources

### Be Focused
- One topic per issue
- Keep discussions on-topic
- Use appropriate labels

## Issue Labels

- `design` - Protocol design discussions
- `security` - Security analysis and concerns
- `performance` - Performance considerations
- `documentation` - Documentation improvements
- `question` - General questions
- `research` - Related research and papers

## What We're NOT Looking For

- Implementation PRs (there's no code to contribute to yet)
- Marketing or promotional content
- Off-topic discussions
- Requests for production deployment

## Code of Conduct

Be professional, respectful, and constructive. Harassment, trolling, or abusive behavior will not be tolerated.

## Questions?

Open an issue with the `question` label or contact: soham@voloweb.io
